# Set-LMLogAlert

## SYNOPSIS

Updates a log alert processor in LogicMonitor.

## SYNTAX

### Id (Default)

```
Set-LMLogAlert -Id <int> [-NewName <string>] [-PipelineId <int>] [-AlertQuery <string>]
 [-AlertType <string>] [-Severity <string>] [-Description <string>] [-LogNotes <string>]
 [-LogMetadataFields <string[]>] [-Disabled <bool>] [-Clear <psobject>] [-Grouping <psobject>]
 [-Window <psobject>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### Name

```
Set-LMLogAlert -Name <string> [-NewName <string>] [-PipelineId <int>] [-AlertQuery <string>]
 [-AlertType <string>] [-Severity <string>] [-Description <string>] [-LogNotes <string>]
 [-LogMetadataFields <string[]>] [-Disabled <bool>] [-Clear <psobject>] [-Grouping <psobject>]
 [-Window <psobject>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObject

```
Set-LMLogAlert -InputObject <psobject> [-NewName <string>] [-PipelineId <int>]
 [-AlertQuery <string>] [-AlertType <string>] [-Severity <string>] [-Description <string>]
 [-LogNotes <string>] [-LogMetadataFields <string[]>] [-Disabled <bool>] [-Clear <psobject>]
 [-Grouping <psobject>] [-Window <psobject>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIPTION

The Set-LMLogAlert function modifies an existing log alert processor in LogicMonitor.
You can specify individual parameters or provide a complete configuration object using InputObject.
When -Disabled is specified, the enable/disable action endpoint is updated separately.

## EXAMPLES

### EXAMPLE 1

Set-LMLogAlert -Id 5 -Description "Updated alert rule"

### EXAMPLE 2

Set-LMLogAlert -Name "High Error Rate" -Disabled $true
